PHASE 2: DOCUMENT INGESTION (Days 2–3)
----------------------------------------------------------------

1. Build FileUploader component:
   - Drag-and-drop using HTML5 drag events (no library needed)
   - Input[type=file] fallback
   - File type validation + size check
   - Upload to Supabase Storage → get public URL

2. Build file parsers in /src/lib/fileParser.ts:
   PDF:  Use pdfjs-dist getDocument() → iterate pages → getText()
   DOCX: Use mammoth.extractRawText() → returns { value: string }
   PPTX: Parse PPTX as zip (JSZip), extract slide XML, parse text nodes
   TXT:  FileReader.readAsText()
   All return: { title: string, rawText: string, htmlContent: string }

3. Build PasteNotes component:
   - Textarea with live word count
   - Detect markdown headings → set title from first H1

4. After parsing: insert row into `documents` table with parsed data

5. Redirect to Studio page with document ID

   - window.speechSynthesis API

   - Functions: play(), pause(), stop(), seek(charIndex), setSpeed(), setPitch()
   - Split document text into sentences (use regex or Intl.Segmenter)
   - Track current sentence index

2. Build TTSController (floating bottom bar):
   - Play/Pause/Stop buttons (Lucide icons)
   - Speed and pitch sliders (Shadcn Slider)
   - Volume slider
   - Current position indicator
   - Keyboard shortcut: Space = play/pause

3. Build VoiceSelector:
   - Load window.speechSynthesis.getVoices()
   - Filter by language (Select dropdown)
   - Voice preview button
   - Save selected voice to user_preferences

4. Highlight current sentence in reader:
   - Wrap each sentence in <span data-sentence-index="N">
   - On sentence change → add .tts-active class
   - CSS: .tts-active { background: rgba(201,168,76,0.25); }
   - Auto-scroll reader to active sentence

PHASE 5: AMBIENT SOUNDS (Days 5–6)
----------------------------------------------------------------

1. Sound assets strategy:
   - Nature sounds: use free CC0 loops from freesound.org or
     generate with Tone.js (preferred — no file hosting needed)
   - Lo-fi beats: short royalty-free loops (host in Supabase Storage)
   - Noise types: generate programmatically with Web Audio API
   - Binaural beats: generate with Tone.js oscillators

2. Implement useAmbientAudio hook:
   - AudioContext with gain nodes per sound
   - Load/decode audio buffers, loop seamlessly
   - Per-sound volume control
   - Global volume control
   - Mix multiple sounds simultaneously (separate AudioBufferSourceNodes)
   - Crossfade on switch (ramp gain up/down over 500ms)

3. Build AmbientPlayer (bottom-right expandable panel):
   - FAB button (Music icon) → expands to panel
   - Grid of sound tiles (icon + label + active indicator)
   - Active sounds show animated glow
   - Per-active-sound volume mini-slider
   - Preset buttons: Deep Focus / Chill Study / Power Study
   - Sleep timer select

4. Sound categories and Tone.js implementations:
   White Noise: Tone.Noise('white').connect(gain)
   Pink Noise:  Tone.Noise('pink').connect(gain)
   Brown Noise: Tone.Noise('brown').connect(gain)
   Binaural (Alpha 10Hz):
     Left:  Tone.Oscillator(200, 'sine')
     Right: Tone.Oscillator(210, 'sine')
     Pan each to L/R channel with Tone.Panner

TECHNICAL NOTES & GOTCHAS
----------------------------------------------------------------

PDF parsing:
  pdfjs-dist requires a worker. Import workerSrc:
  import * as pdfjsLib from 'pdfjs-dist'
  pdfjsLib.GlobalWorkerOptions.workerSrc = 
    `//cdnjs.cloudflare.com/ajax/libs/pdf.js/${pdfjsLib.version}/pdf.worker.min.js`

PPTX parsing (no great npm library):
  Option A: Use JSZip to unzip the .pptx, then parse slide XML with DOMParser
  Option B: Send to Claude API with file content as base64 and ask it to extract text
  Option C: Supabase Edge Function calling LibreOffice headless

Web Speech API gotcha:
  getVoices() is async in Chrome. Listen to 'voiceschanged' event:
  window.speechSynthesis.addEventListener('voiceschanged', loadVoices)

Audio Context policy:
  AudioContext must be created after user gesture (click).
  Lazy-init AudioContext on first play button press.

Claude API from browser:
  Include API key in request headers. For production, proxy through
  a Supabase Edge Function to hide the key from client.

Supabase realtime (optional stretch):
  Could use Realtime to sync highlights/notes across devices.

Large documents:
  If rawText > ~100k chars, chunk before sending to Claude.
  Use a sliding window approach or summarize in segments.
